# イベント参加フォーム機能 仕様書

**作成日:** 2026-05-24  
**ステータス:** Draft

---

## 1. 概要

既存のアンケート機能を拡張し、オフ会などのイベント参加募集・管理に対応する。  
アンケート作成画面の既存UIに「イベント参加フォームとして作成する」チェックボックスを追加することで、部制対応の参加管理機能が有効になる。

> [!IMPORTANT]
> 応募者の特定・通知（DM送信）に Discord ユーザーID を使用するため、  
> Discord ログインなしではフォームを送信できません。  
> フォーム冒頭にもこの旨を明示表示します。

---

## 2. 用語定義

| 用語 | 説明 |
|---|---|
| イベント | アンケートに紐づく開催情報（日程・場所・参加費） |
| 部 (session) | イベント内の時間帯区分。0件なら部制なし |
| 参加者 (participant) | アンケートに回答したユーザー |
| オーナー | イベントを作成・管理するユーザー |

---

## 3. 機能要件

### 3.1 アンケート作成（拡張）

- アンケート作成画面に「イベント参加フォームとして作成する」チェックボックスを追加する
- チェック時、以下の追加入力欄が展開される：
  - 参加費（円）※ 任意
  - 部の追加（0件以上）
    - 部名（例: "1部", "2部"）
    - 日時
    - 集合場所
    - 定員（人数。空欄 = 無制限）
- 参加可否・希望部の選択肢はシステムが**自動生成**する（オーナーが手書き不要）

#### 自動生成される質問

| 部の有無 | 自動生成質問 | 選択肢 |
|---|---|---|
| 部なし | 「参加しますか？」 | はい / いいえ |
| 部あり | 「参加希望の部を選択してください」 | 1部 / 2部 / … / 不参加 |

### 3.2 参加者の応募

- 通常のアンケートと同じ画面・操作感で回答する
- 回答送信をもって参加応募とみなす
- **Discord ログインユーザーのみ応募可能**（user_id を紐づけ）
- フォーム冒頭に以下を常時表示する：
  ```
  ⚠️ このフォームへの応募には Discord アカウントが必要です。
  未ログインの場合、送信ボタンは無効になります。
  ```
- 未ログイン状態でアクセスした場合は送信ボタンを非活性にし、Discord ログインへ誘導する

### 3.3 割り当てロジック（自動）

オーナーが「自動割り当て実行」を行うと以下の順で処理する：

```
1. 「不参加」回答者 → 対象外
2. 希望部に空きあり → 承認・確定
3. 希望部が満席 → 他の空き部に振り分け
4. 全部満席 → 補欠（waitlist）に登録
5. 部なしの場合 → 全員承認（定員なければ上限なし）
```

- 「両方希望」（複数部選択）の場合は第一希望優先、満席時は第二希望へ
- 自動割り当て後もオーナーが手動で上書き可能

### 3.4 オーナー管理UI

#### 応募一覧画面

- 応募者一覧を表示（名前・希望部・割り当て状況）
- 各参加者に対して以下の操作が可能：
  - 部の手動変更（ドロップダウン）
  - 承認 / 否認 / 補欠 の手動設定
  - 個人メモの入力（参加者には非表示）
- 部ごとの残席数をリアルタイム表示
- 「自動割り当て実行」ボタン
- 「全員に通知送信」ボタン（確定後に有効化）

### 3.5 締切設定

- イベント作成時に `application_deadline`（応募締切日時）を任意で設定できる
- 締切を過ぎたフォームは送信ボタンを非活性にし「受付終了」を表示する
- `NULL` の場合は無期限受付

### 3.6 通知

#### Discord DM（Bot 経由）

確定後、オーナーが「通知送信」を実行すると Bot が各参加者に DM を送信する。

**承認時のDM例：**
```
【オフ会 参加確定のお知らせ】
━━━━━━━━━━━━━━━
✅ 1部 参加確定
📅 2024/XX/XX 13:00〜
📍 〇〇駅 XX番出口
💴 参加費: 3,000円

📆 カレンダーに追加:
・Google: https://calendar.google.com/calendar/render?...
・Outlook: https://outlook.live.com/calendar/deeplink/compose?...

詳細確認: https://.../event/confirm/abc123
━━━━━━━━━━━━━━━
```

**否認時のDM例：**
```
【オフ会 参加について】
申し訳ございませんが、今回は参加をお断りさせていただきます。
またの機会にぜひご参加ください。
```

- カレンダーURLはOAuth不要・URLパラメータのみで生成（Google / Outlook / .ics）
- カレンダーリンクは承認DMと個人確認ページの両方に掲載する
- 通知済みフラグを管理し、二重送信を防止
- 部や承認状況が変更された場合は「再通知」が可能

### 3.6 個人確認ページ

- URL: `/event/<token>` （ランダムトークン）
- ログイン不要でアクセス可能（トークンが本人確認代わり）
- 表示内容：
  - 参加確定状況（確定 / 否認 / 補欠 / 確認中）
  - 割り当てられた部・日時・場所・参加費
  - オーナーからの個人メモ（あれば）

## 4. データ設計

### events

| カラム | 型 | 説明 |
|---|---|---|
| id | INTEGER PK | |
| survey_id | INTEGER FK | アンケートへの参照 |
| title | TEXT | イベント名 |
| fee | INTEGER | 参加費（円）。null = 無料 |
| notes | TEXT | 全体向け備考 |
| status | TEXT | `draft` / `open` / `closed` |
| created_at | DATETIME | |

### event_sessions（部）

| カラム | 型 | 説明 |
|---|---|---|
| id | INTEGER PK | |
| event_id | INTEGER FK | |
| name | TEXT | 部名（例: "1部"） |
| datetime | DATETIME | 集合日時 |
| location | TEXT | 集合場所 |
| capacity | INTEGER | 定員。null = 無制限 |

※ event_sessions が 0件 → 部制なし

### event_participants（個人の参加状況）

| カラム | 型 | 説明 |
|---|---|---|
| id | INTEGER PK | |
| event_id | INTEGER FK | |
| user_id | INTEGER | Discord user_id |
| response_id | INTEGER FK | アンケート回答への参照 |
| session_id | INTEGER FK | 割り当て部。null = 不参加 |
| preferred_session_ids | TEXT | 希望部IDのJSON配列 |
| approval | TEXT | `pending` / `accepted` / `rejected` / `waitlist` |
| personal_note | TEXT | オーナーの個人メモ |
| access_token | TEXT | 個人確認ページ用トークン |
| notified_at | DATETIME | DM送信日時。null = 未送信 |

## 5. 画面一覧

| 画面 | URL | 対象 |
|---|---|---|
| アンケート作成（拡張） | `/survey/create` | オーナー |
| 応募一覧・管理 | `/event/<event_id>/admin` | オーナー |
| 個人確認ページ | `/event/<token>` | 参加者 |

---

## 6. 実装ステップ

| Step | 内容 |
|---|---|
| 1 | DBスキーマ追加（events / event_sessions / event_participants） |
| 2 | アンケート作成画面にチェックボックス・部入力UI追加 |
| 3 | 応募受付ロジック（回答送信 → event_participants 登録） |
| 4 | 自動割り当てロジック実装 |
| 5 | Admin管理UI（応募一覧・手動操作） |
| 6 | Discord DM通知（Bot連携） |
| 7 | 個人確認ページ |

## 7. 未決定事項
