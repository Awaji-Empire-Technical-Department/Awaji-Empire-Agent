# バグ修正仕様書: フォームラジオボタン name 属性の衝突

- **バグID**: BUGFIX-FORM-RADIO-001
- **発見日**: 2026-02-23
- **作業ブランチ**: `fix/form-radio-name-conflict`
- **影響ファイル**: `discord_bot/templates/form.html`
- **関連ファイル**: `discord_bot/static/js/form.js`（新規作成）

---

## 1. 症状

| # | 症状 | 再現条件 |
|---|------|----------|
| 1 | Q1 で「はい」と「いいえ」を **両方同時に選択** できる | ラジオボタン質問が1つ以上あるとき |
| 2 | 他の質問を選択すると **Q1 の選択状態が崩れる** | 複数ラジオボタン質問があるとき |
| 3 | 条件分岐(Branch Logic)が **一切機能しない** | `logic` 付き質問があるとき |

---

## 2. 根本原因

### Jinja2 ネストループでの `loop` 変数スコープ上書き問題

`form.html` において、外側ループ変数 `loop.index0`（＝質問番号）を内側の
選択肢ループが **上書き** していた。

```jinja2
{# 外側ループ: 質問 #}
{% for q in questions %}
    {% set q_idx = loop.index0 %}   {# ← ここでは正しく取れる (0, 1, 2...) #}

    {# 内側ループ: 選択肢 #}
    {% for opt in q.options %}
        {# loop が内側ループに上書きされた状態 #}
        {# loop.index0 = 0, 1, 2... (選択肢番号) #}
        <input type="radio" name="q_{{loop.index0}}" ...>  {# ← ❌ バグ #}
    {% endfor %}
{% endfor %}
```

### name 属性のズレ（Q1・Q2が「はい/いいえ」2択の場合）

| 要素 | 正しい name | 実際の name（バグ） |
|------|------------|-------------------|
| Q1「はい」(選択肢0) | `q_0` | `q_0` ← 偶然合う |
| Q1「いいえ」(選択肢1) | `q_0` | `q_1` ← **ズレる** |
| Q2「はい」(選択肢0) | `q_1` | `q_0` ← **Q1と混在** |
| Q2「いいえ」(選択肢1) | `q_1` | `q_1` ← **Q1いいえと混在** |

### checkLogic() への連鎖影響

```javascript
// name="q_X" の X をキーとして回答を収集する
ans[r.name.split('_')[1]] = r.value;

// Q2の条件: data-trg="0" (Q1のインデックス)
// → ans["0"] を参照するが、バグにより ans には"0"が存在しないケースがある
// → 条件分岐が常に評価不能になる
```

---

## 3. 修正方針

Jinja2 の**外側ループで保存した変数 `q_idx`** を使用する。
この変数は `{% set q_idx = loop.index0 | string %}` で正しく取得済みのため、
内側ループ内でも安全に参照できる。

```jinja2
{# 修正後 #}
<input type="radio" name="q_{{q_idx}}" ...>  {# ✅ q_idx は外側ループの値 #}
```

### 修正箇所（form.html）

| 行 | 対象 | 修正内容 |
|----|------|----------|
| 44 | radio `name` | `loop.index0` → `q_idx` |
| 52 | radio その他 `name` | `loop.index0` → `q_idx` |
| 65 | checkbox `name` | `loop.index0` → `q_idx` |
| 77 | checkbox その他 `name` | `loop.index0` → `q_idx` |

---

## 4. 追加対応: JS の外部ファイル化

`form.html` 内のインライン `<script>` ブロック（`checkLogic`, `toggleOther`,
`toggleReq` 関数）を **`discord_bot/static/js/form.js`** として分離する。

### 理由

- ロジックとビューの分離（保守性向上）
- ブラウザキャッシュによるパフォーマンス向上
- テスト・デバッグの容易化
