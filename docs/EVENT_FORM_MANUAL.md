## 1. 管理者向け：イベントを作成する

### 1-1. フォーム作成画面を開く

1. ダッシュボードの **「アンケート作成」** ボタンをクリック
2. フォームのタイトル・質問を入力する

### 1-2. イベント参加フォームとして設定する

フォーム作成画面の下部にある **「イベント参加フォームとして作成する」** にチェックを入れると、イベント設定エリアが展開されます。

#### 入力項目（共通）

| 項目 | 必須 | 説明 |
|---|---|---|
| イベント名 | ○ | 通知DMやカレンダーリンクに使用されます |
| 参加費 | — | 円単位で入力。0または空欄 = 無料 |
| 応募締切 | — | 設定すると期限後にステータスが自動で「締切」に変わります |
| 備考 | — | フォームに表示される補足情報 |

#### 部制なし（1回開催）の場合

| 項目 | 必須 | 説明 |
|---|---|---|
| 開催日時 | — | カレンダーリンク生成に使用。空欄可 |
| 終了日時 | — | 空欄時は開始+2時間でカレンダーURL生成 |
| 集合場所 | — | カレンダーリンクに含まれます |
| 定員 | — | 空欄 = 無制限。設定すると自動割り当て時に上限を考慮します |

#### 部制あり（複数回開催）の場合

「部を追加」ボタンで部を追加できます。各部に以下を設定します。

| 項目 | 必須 | 説明 |
|---|---|---|
| 部名 | ○ | 例：「1部」「午前の部」 |
| 開催日時 | — | 部ごとに個別設定可 |
| 終了日時 | — | 空欄時は開始+2時間 |
| 集合場所 | — | 部ごとに異なる場所を設定可 |
| 定員 | — | 空欄 = 無制限 |

### 1-3. フォームを公開する

1. 「保存して公開」をクリック
2. ダッシュボードに表示される **フォームURL** を Discord などで告知する
3. ステータスが **「受付中」** になっていることを確認する

> **注意:** 応募者はDiscordログインが必要です。フォームURLを共有する際にその旨を案内してください。

## 2. 管理者向け：応募管理・割り当てを行う

### 2-1. イベント管理画面を開く

ダッシュボードのアンケート一覧から対象フォームの **「イベント管理」** ボタンをクリックします。

### 2-2. 応募者一覧を確認する

画面下部の表に応募者が一覧表示されます。

| 列 | 説明 |
|---|---|
| 名前 | Discordユーザー名 |
| 希望 | 応募者が選択した希望部 |
| 割り当て部 | 管理者が割り当てた部（手動変更可） |
| 状態 | 確認中 / 承認 / 否認 / 補欠 |
| 個人メモ | 参加者には非公開のメモ欄 |
| 通知 | DM送信済みかどうか |

### 2-3. 自動割り当てを実行する

**「自動割り当て実行」** ボタンを押すと、以下のルールで割り当てが行われます。

- 希望部を優先して割り当て
- 定員に達した部には補欠（waitlist）として登録
- 既に **承認・否認** 済みの方はスキップされます（上書きしません）

> 実行後はDOMがリアルタイムで更新されます。確認後に通知を送信してください。

### 2-4. 手動で調整する

自動割り当て後も、表の各セルで **割り当て部** と **状態** を直接変更できます。変更は即時保存されます。

### 2-5. CSVでダウンロードする

**「CSV ダウンロード」** ボタンで全応募データをExcelで開けるCSV形式で取得できます。

イベントフォームの場合、以下の列が追加されます：

| 列名 | 内容 |
|---|---|
| 参加意思 | 参加 / 不参加 |
| 状態 | 確認中 / 承認 / 否認 / 補欠 |
| 割り当て部 | 割り当て済みの部名 |
| 希望部 | 応募者が選択した希望部（複数の場合はカンマ区切り） |

## 3. 管理者向け：参加通知を送信する

### 3-1. 通知を送る前の確認

- 承認・否認・補欠 の割り当てが完了していること
- 状態が「確認中」のまま送信すると、その方への通知はスキップされます

### 3-2. 一斉通知を送信する

**「全員に通知送信」** ボタンをクリックすると、承認・否認・補欠の全員に Discord DM が送信されます。

- **送信済みの方はスキップされます**（二重送信なし）
- 送信後は通知列に「送信済み」が表示されます

### 3-3. 送信されるDMの内容

#### 承認（参加確定）の場合

```
【イベント名】参加確定のお知らせ
━━━━━━━━━━━━━━━
✅ 1部 参加確定
📅 2026-05-31 14:00:00
📍 集合場所
💴 参加費: 1000円

📆 カレンダーに追加:
・Google: <Google CalendarのURL>
・Outlook: <OutlookのURL>
━━━━━━━━━━━━━━━
詳細確認: https://dashboard.awajiempire.net/event/confirm/...
```

#### 補欠の場合

```
【イベント名】補欠登録のお知らせ
━━━━━━━━━━━━━━━
⏳ 補欠として登録されました。
キャンセルが出た場合にご連絡します。
━━━━━━━━━━━━━━━
詳細確認: ...
```

#### 否認の場合

```
【イベント名】参加見送りのお知らせ
━━━━━━━━━━━━━━━
❌ 今回は参加見送りとなりました。
またの機会にぜひご参加ください。
━━━━━━━━━━━━━━━
詳細確認: ...
```

### 3-4. 個別に確認URLをコピーする

表の右端の **リンクアイコン** ボタンで、個別の結果確認URLをクリップボードにコピーできます。DMが届かなかった方への手動連絡に使用してください。

## 4. 参加者向け：フォームに応募する

### 4-1. Discordでログインする

フォームURLを開くと、まず **「Discordでログイン」** ボタンが表示されます。  
ログインしないとフォームを送信できません。

> ログインしてもサーバーへの権限付与などは行いません。ユーザーIDと名前の取得のみです。

### 4-2. 希望部を選ぶ（部制ありの場合）

部ごとに以下の情報が表示されます：

- 部名（例：「1部」「午前の部」）
- 開催日時
- 集合場所
- 残席数（残り3席以下で表示。満席の部は選択不可）

希望する部を1つ選んでください。

### 4-3. アンケートに回答して送信する

通常のアンケートと同様に質問に回答し、**「送信」** ボタンをクリックします。

> 「不参加（辞退）」を選択した場合もフォームを送信してください。管理者への意思表示になります。

### 4-4. 応募完了

送信後、応募完了ページが表示されます。結果（承認・否認・補欠）はDiscord DMでお知らせします。

## 5. 参加者向け：結果を確認する

### 5-1. DMで結果を受け取る

管理者が通知を送信すると、Discord DM が届きます。  
承認の場合はカレンダー登録リンクが含まれています。

### 5-2. 確認ページで詳細を見る

DMに記載の **「詳細確認」URL** を開くと、現在の参加状況をいつでも確認できます。

| 表示内容 | 説明 |
|---|---|
| 参加状態 | 確認中 / 参加確定 / 補欠 / 参加見送り |
| 割り当て部 | 確定した参加部（承認時のみ） |
| カレンダー追加 | Google Calendar / Outlook / .ics ダウンロード |

## 6. よくある質問

**Q. 応募後に部の希望を変更できますか？**  
A. フォームの再送信はできません。変更が必要な場合はイベント主催者に直接連絡してください。

**Q. 満席の部には応募できませんか？**  
A. 満席の部は選択できません。他の部を選ぶか、主催者に補欠登録を相談してください。

**Q. DMが届きません。**  
A. Discordのプライバシー設定で「サーバーメンバーからのDM」が許可されているか確認してください。それでも届かない場合は主催者から個別の確認URLを共有してもらってください。

**Q. カレンダーリンクの時刻がずれています。**  
A. カレンダーリンクはJSTローカル時刻で生成されています。お使いのカレンダーアプリのタイムゾーン設定を確認してください。

**Q. 「不参加」で送信したのに参加扱いになっています。**  
A. 「不参加」選択は管理者への意思表示です。最終的な承認・否認は管理者が手動または自動割り当てで決定します。

**Q. 参加費の事前決済（オンライン決済）を導入したいです。**  
A. 現在は参加費の表示のみ対応しています。Stripe等による事前決済が必要な場合は **Wanyaldee** までご連絡ください。
