# Architecture Decision Records (ADR)

このディレクトリは、Awaji Empire Agent プロジェクトにおける重要なアーキテクチャ上の意思決定を記録します。

## ディレクトリ構成

```
docs/adr/
├── README.md          ← このファイル（インデックス・運用ルール）
├── TEMPLATE.md        ← 新規 ADR 作成用テンプレート
├── 001-010/           ← ADR-001〜010: Phase 2〜4 基盤アーキテクチャ
│   └── README.md
├── 011-020/           ← ADR-011〜020: ラウンジ・大会・称号システム
│   └── README.md
├── 021-030/           ← ADR-021〜030: イベントシステム・インフラ整備
│   └── README.md
└── 031-040/           ← ADR-031〜040: （031 作成時に新設）
    └── README.md
```

## 運用ルール

### 1. 番号付けとディレクトリ

- ADR 番号は **連番** で採番する（例: ADR-024, ADR-025, …）
- **10件ごと**にディレクトリを切る
  - ADR-001〜010 → `001-010/`
  - ADR-011〜020 → `011-020/`
  - ADR-021〜030 → `021-030/`
  - ADR-031〜040 → `031-040/`（ADR-031 が発生したときに作成）
- 新しい 10件ブロックに入った際は、そのディレクトリと `README.md` を **必ず同時に作成** する

### 2. 新規 ADR の作成手順

1. 次の番号を確認する（`021-030/` 内の最大番号 + 1）
2. `TEMPLATE.md` をコピーして `NNN-kebab-case-title.md` にリネーム
3. 対応するディレクトリ（`021-030/` など）に配置
4. そのディレクトリの `README.md` の一覧表に追記する

### 3. ステータスの種類

| ステータス | 意味 |
|-----------|------|
| 提案中 (Proposed) | 議論中・未確定 |
| 採用 (Accepted) | 採用が確定し実装済みまたは実装予定 |
| 承認済み (Approved) | レビュー・承認を経た確定済み |
| 廃止 (Deprecated) | 後続の ADR により置き換えられた |

### 4. ADR を書くタイミング

以下のような意思決定を行った際に記録する。

- アーキテクチャの新規採用・変更
- 技術スタックの選定・移行
- セキュリティ・運用上の重要な方針変更
- 既存設計の大幅な修正や廃止
