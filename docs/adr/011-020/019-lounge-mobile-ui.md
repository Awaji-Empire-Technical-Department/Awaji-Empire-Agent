# ADR-019: ラウンジ UI モバイル対応修正・WS 切断時のポーリング補完

- **ステータス**: 採用
- **作成日**: 2026-05-23
- **作成者**: Wanyaldee
- **関連ADR**: [ADR-018](018-lounge-final-score-reporting.md)

---

## コンテキスト

Phase 3 の申告方式変更（ADR-018）に伴い、ラウンジ UI を全面刷新した。
その過程で以下の2つの問題が残存していた。

### 問題1: モバイルでボタンレイアウトが崩れる

`lounge.css` のモバイル向けセレクタが `#staff-panel` を参照していたが、
HTML リファクタリングで同要素が `#action-panel` にリネームされていたため、
モバイル用スタイルが一切当たらない状態になっていた。

具体的な症状:
- アクションパネルのボタンが横幅いっぱいに広がらない
- 最終順位入力の select + ボタン行がモバイル幅で横にはみ出る
- 除外ボタンのタッチターゲットが小さすぎる（32px 未満）
- 結果モーダルの MMR グリッドが狭い画面で窮屈

### 問題2: ページリロード・WS 切断中の状態消失

- **リロード後**: `loadFinalScores()` は呼ばれていたが、WS 再接続後に再取得しないため切断中の差分が反映されない
- **WS 切断中**: 他プレイヤーの申告が届かず、手動リフレッシュが必要だった

---

## 決定内容

### モバイル CSS 修正

1. **セレクタ修正**: `#staff-panel` → `#action-panel`
2. **タッチターゲット確保**: `.btn` の `min-height: 48px`（Apple HIG 準拠）
3. **`final-rank-row` クラス新設**: select + ボタン行をモバイルで縦並びに切り替え
4. **`exclude-btn` クラス新設**: 動的生成する除外ボタンに一貫したサイジングを適用
5. **結果モーダル**: `grid-2col` クラスでモバイル時に縦並び化

### WS 切断補完（ポーリング）

| タイミング | 処理 |
|-----------|------|
| WS `close` イベント | `startPolling()` — 10秒ごとに `loadFinalScores()` を実行 |
| WS `open` イベント（再接続） | `stopPolling()` + `loadFinalScores()` で切断中の差分を一括同期 |
| ページリロード | 初期ロード時の `loadFinalScores()` で DB から状態を復元（既存動作） |

ポーリング間隔を 10 秒に設定した根拠:
- WS 切断はまれなケースのため頻繁なポーリングは不要
- 30 秒だと申告完了の通知が遅れすぎて UX を損なう
- 10 秒は実用的な妥協点
