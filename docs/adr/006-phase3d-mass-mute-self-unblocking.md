# ADR-006: Phase 3-D — サーバー権限を利用した Bot 自身の権限復旧（Self-unblocking）の導入

- **日付**: 2026-02-25
- **ステータス**: 承認済み
- **ブランチ**: `feature/phase3d-hotfix`

---

## 1. 背景

ADR-005 に基づく Phase 3-D ホットフィックスの実装過程にて、マスミュート機能の「自己修復（Self-Healing）」において新たな課題が浮上した。

Bot がサーバー全体の「ロールの管理（Manage Roles）」権限を取得していても、特定のチャンネルの「チャンネル設定 > 権限」において、Bot のロールや Bot 自身が明示的に拒否（❌）設定されている場合、`Forbidden` エラーが発生し、そのチャンネルへのミュート適用や修復が停止してしまった。

本番環境の Discord サーバーは外部の管理者が所有しており、Bot が自らこの制限を突破できないと、可用性の維持が管理者の個別設定スキルに依存してしまう。

---

## 2. 決定事項 — セルフ・アンブロッキング (Self-unblocking)

`Forbidden` エラーを検知した際、Bot が以下のロジックで自律的に復旧する「セルフ・アンブロッキング」機能を導入した。

1. **サーバー権限の確認**: Bot がサーバーレベルの「ロールの管理」権限を保有しているか確認する。
2. **自身の制限解除**: 権限を保有している場合、そのチャンネルの「メンバー設定（Member Overwrites）」として Bot 自身を登録し、「権限の管理（Manage Permissions）」を強制的に許可（✅）に上書きする。
3. **リトライ**: 自身のブロックを解除した状態で、本来のミュート適用（@everyone への上書き）を再試行する。

---

## 3. 採用理由

1. **可用性の自律保証**: 管理者によるチャンネル設定の手動修正を待つことなく、Bot が自らの実行環境を正常化できる。
2. **運用コストの最小化**: 「サーバーレベルで権限を付与する」という一度の設定だけで、各チャンネルごとのパーミッション設定の矛盾を Bot が自動で解決できる。
3. **明確なエラーフィードバック**: 万が一サーバー全体の権限すら不足している場合は、追加された事前チェックロジック（ADR-005 参照）により、管理者に正確な是正アクションを DM 通知する。

---

## 4. 影響

- `services/permission_service.py` に `repair_self_if_blocked()` メソッドを実装。
- `apply_permission` および `check_and_repair` の `except discord.Forbidden` 節でこのメソッドを呼び出し、自動リトライを行うよう変更。
- 診断コマンド `!mute_check` も、このチャンネル固有のブロックを検知し、詳細に報告できるよう強化。
