# ラウンジシステム 運営ガイド

> 対象: ラウンジ運営に関わるスタッフ・ホスト・参加者全員  
> 最終更新: 2026-05-23（Phase 3 対応）

---

## 1. システム概要

マリオカートワールドのラウンジ運営を補助するWebシステムです。  
ゲーム内でのレース進行はゲーム本体に任せ、**参加管理・MMRレート計算・称号付与**をWebアプリで行います。

| 作業 | 担当 |
|------|------|
| ルーム作成・CPU設定・レース進行 | ゲーム本体（マリオカートワールド） |
| 参加受付・最終順位申告・MMR計算 | **Webアプリ（本システム）** |
| ランク称号の付与・Discordロール同期 | **Webアプリ → Discord自動** |

---

## 2. ラウンジの流れ（全体）

```
[ホスト]                          [参加者]
   |                                 |
セッション作成                        |
   |                                 |
ゲーム内ルーム作成                     |
   |                                 |
ルームID告知 ──────────────────→ ゲームに入室
   |                                 |
         12レースをプレイ
   |                                 |
「セッション終了」ボタン押下            |
   ↓                                 ↓
最終順位申告モーダルが全員に表示
   |                                 |
各自が最終順位（1〜24位）を申告 ←─── 申告する
   |                                 |
ホストが「終了確定・MMR反映」を押す    |
   ↓                                 ↓
  MMR計算・称号付与・結果モーダル表示
   |                                 |
         ダッシュボードに戻る
```

## 3. ホスト操作手順

### 手順1: セッションを作成する

1. [Webダッシュボード](/) にログインし、上部メニューから **「ラウンジ」** をクリック
2. 右上の **「セッション作成」** ボタンをクリック
3. 以下を設定して **「作成」** を押す

| 項目 | 内容 |
|------|------|
| モード | FFA（個人戦）/ 2v / 3v |
| レース数 | 通常12（変更可） |
| ルームID | ゲーム画面に表示される6桁英数字（例: `ABC123`） |

> セッション作成後、ダッシュボードに「ラウンジ募集中！」バナーが表示されます。

### 手順2: ゲーム内でルームを立てる

ゲーム本体でオンラインルームを作成してください。推奨設定：

| 設定 | 値 |
|------|----|
| CPU強さ | 強い |
| アイテム | ノーマル |
| インターバル | 10秒 |
| 最大人数 | 24人 |

### 手順3: 12レースをプレイ

レース中のWebアプリへの操作は不要です。ゲームに集中してください。  
参加者はWebアプリを開いておくだけでOKです（自動で参加者一覧に表示されます）。

### 手順4: セッションを終了する

12レース終了後：

1. セッション画面で **「セッション終了（確定）」** ボタンをクリック
2. 最終順位申告モーダルが開く（全員に同時表示）
3. 自分の最終順位を選んで **「申告する」** を押す
4. 他の参加者の申告状況をリアルタイムで確認できる
5. 申告できないプレイヤーがいる場合は **「除外」** ボタンで対象外にできる
6. 全員が申告したら（または除外後に）**「終了確定・MMR反映」** ボタンを押す

> 終了確定を押すと MMR が即座に計算・更新され、結果モーダルが全員に表示されます。

## 4. 参加者操作手順

### 手順1: セッションに参加する

1. ダッシュボードの「ラウンジ募集中！」バナーで **「参加する」** をクリック
2. または上部メニューの「ラウンジ」から開催中セッションに入場する
3. セッション画面を開いた時点で自動的に参加者一覧に登録される

### 手順2: ゲームに入室する

セッション画面に表示されているルームIDをゲームに入力して入室してください。

### 手順3: 最終順位を申告する

12レース終了後、ホストが終了操作を行うと自動でモーダルが表示されます。

1. ゲームの最終結果画面で**自分の最終順位**を確認する
2. モーダルのセレクトから**自分の順位**を選ぶ（1〜24位）
3. **「申告する」** ボタンを押す

> 申告はセッション中いつでも可能です。「最終順位申告する」ボタンから手動で開けます。

### 手順4: 結果を確認する

全員の申告完了後、ホストが終了確定を押すと**結果モーダルが自動表示**されます。

- **最終順位**（セッション内での確定順位）
- **MMR増加**（今回のセッションで増えたMMR）
- **現在のMMR**（累計）
- **現在のランク称号**

10秒後にダッシュボードへ自動で戻ります。

## 5. MMRレートと称号

### MMRデルタテーブル（1セッションあたりの増加量）

| 順位 | MMR増加 | 順位 | MMR増加 |
|------|--------|------|--------|
| 1位 | **+150** | 13位 | +40 |
| 2位 | +120 | 14位 | +35 |
| 3位 | +100 | 15位 | +30 |
| 4位 | +90 | 16位 | +25 |
| 5位 | +80 | 17位 | +20 |
| 6位 | +75 | 18位 | +20 |
| 7位 | +70 | 19位 | +15 |
| 8位 | +65 | 20位 | +15 |
| 9位 | +60 | 21位 | +10 |
| 10位 | +55 | 22位 | +10 |
| 11位 | +50 | 23位 | +5 |
| 12位 | +45 | 24位 | **+5（参加賞）** |

> 全順位プラスのみ。ラウンジに参加するだけでMMRが上がります。

### ランク称号（MMR累計ベース）

称号はダッシュボードの称号管理から Staff が設定します。  
MMRが閾値に到達した時点でセッション終了後に自動付与されます。

### 優勝称号

セッション1位のプレイヤーには **優勝回数ベースの称号**（例: 覇者）が自動付与されます。

## 7. チーム戦（2v / 3v）

- セッション作成時にモードを「2v」または「3v」に設定する
- ホストが Webアプリ上でチームを編成し、各チームにタグを設定する
- 参加者はゲーム内のプレイヤー名先頭にタグを付ける（例: `[ABC] プレイヤー名`）
- チームスコアは個人スコアの合算で決定（最終順位申告は個人ごとに行う）

## 8. スクリーンショット保存のお願い

> 📸 **ゲームの結果画面は必ずスクリーンショットで保存してください。**

申告内容に異議が生じた場合、スクリーンショットを証拠として提出していただく場合があります。  
Webアプリのセッション画面にも常時リマインダーが表示されます。

## 9. よくある質問

**Q. ゲーム内でルームIDはどこで確認できますか？**  
A. ゲーム本体のオンラインルーム待機画面に表示されます。6桁の英数字です。

**Q. 申告を間違えた場合は？**  
A. 終了確定前であれば、もう一度申告し直すと上書きされます。確定後の修正は Staff に連絡してください。

**Q. 接続が切れてモーダルが表示されなかった場合は？**  
A. セッション画面をリロードすると結果が表示されます。またはダッシュボードから MMR の変化を確認できます。

**Q. 接続問題で申告できなかった場合は？**  
A. ホストに伝えると「除外」扱いにできます（MMR増加なし・セッション参加カウントのみ）。

**Q. Webアプリにアクセスするには？**  
A. Discordアカウントでログインします。淡路帝国サーバーに参加済みであることが条件です。

**Q. 称号はいつ反映されますか？**  
A. セッション終了と同時に自動計算されます。Discord のロールは数秒以内に付与されます。
